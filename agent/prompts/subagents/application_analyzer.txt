**EXTERNAL LIBRARY DEPENDENCY ANALYSIS — REQUIRED**

You MUST thoroughly document ALL third-party package dependencies. This is one of the most
important parts of the analysis. Do NOT skip or summarize this section.

Steps:
1. Read the manifest file (Cargo.toml, package.json, pyproject.toml, go.mod) to get the
   complete list of third-party dependencies with their version constraints.
2. For EACH external dependency, document:
   - **Name**: Exact package name
   - **Version**: Version constraint from the manifest
   - **Category**: Classify as one of: web-framework, database, serialization, async-runtime,
     crypto, networking, logging, testing, cli, build-tool, cloud-sdk, blockchain, monitoring, other
   - **Purpose**: 1-2 sentences on what this package provides and HOW this application uses it
   - **Integration points**: Which modules/files import or use this dependency
3. Use Grep to find actual usage patterns in the codebase
4. Distinguish between runtime dependencies and dev dependencies

Format in the markdown output as:

```
## Dependencies

### External Libraries

- **actix-web** (4.0) [web-framework]: HTTP framework powering the REST API.
  Configures routes, middleware, and request handlers.
  Imported in: `src/main.rs`, `src/routes/`.

- **sqlx** (0.7) [database]: Async PostgreSQL driver with compile-time query validation.
  Used for all database operations via connection pool.
  Imported in: `src/db/`, `src/models/`.

- **serde** (1.0) [serialization]: JSON serialization for API request/response bodies.
  Used via derive macros on all API types.
  Imported in: `src/types/`, `src/api/`.
```

Do NOT output bare dependency names without version, category, and usage details.

- Cloud platforms (GCP Cloud Build, GCP Pub/Sub, GCS, AWS S3, AWS SQS)

- Payment APIs (Stripe, Braintree)
- Auth/security services (Auth0, Intel Trust Authority, AWS Cognito)
- Message queues (RabbitMQ, Kafka, Google Pub/Sub)
- Any other external network-accessible system

Format each as a named entry with description:
```
## External Systems
- **PostgreSQL**: Primary relational database — stores app metadata, sessions, build records
- **GCP Cloud Build**: Triggers verifiable container builds; reads build logs via API
- **GCP Pub/Sub**: Subscribes to instance lifecycle events (instance-events topic)
- **Ethereum RPC**: Reads smart contract events via JSON-RPC; submits transactions
- **Stripe**: Payment processing API — subscription creation, webhook verification
```
