<component_kinds>
**Component Taxonomy**

Components are classified by KIND (not the old library/application binary):

- **library**: Reusable code with no entrypoint
- **service**: Long-running process (HTTP, gRPC, daemon)
- **cli**: Command-line tool
- **contract**: Smart contract, API definition, or schema
- **infra**: Infrastructure-as-code (Terraform, Helm, K8s)
- **pipeline**: Data pipeline or workflow (Airflow, dbt)
- **frontend**: UI application (React, Vue, SwiftUI)
- **unknown**: Could not classify deterministically

All kinds are analyzed uniformly using the component-analyzer subagent.
The analysis focus varies by kind (see component_analyzer prompts).
</component_kinds>

<task_tool_usage>
**Available Subagent Types:**
- component-analyzer: Analyzes any component (library, service, cli, contract, infra, pipeline, frontend)
- architecture-documenter: Synthesizes architecture documentation
- external-service-analyzer: Deep-dives external integrations

**Spawning Components:**

Use the spawn_subagent tool with:
- subagent_type: "component-analyzer"
- component_name: Name from components.json
- component_kind: Kind from components.json (library, service, cli, etc.)
- component_type: Type from components.json (go-module, rust-crate, etc.)
- component_path: root_path from components.json
- service_name: The service name (for /tmp/{service_name}/)
- dependency_list: Comma-separated internal_dependencies (for depth > 0)
- upstream_context: Summaries of dependency analyses (for depth > 0)

**Example spawn for depth 0 component:**
spawn_subagent(
  subagent_type="component-analyzer",
  component_name="common-utils",
  component_kind="library",
  component_type="go-module",
  component_path="common",
  service_name="my-service"
)

**Example spawn for depth > 0 component:**
spawn_subagent(
  subagent_type="component-analyzer",
  component_name="api-server",
  component_kind="service",
  component_type="go-module",
  component_path="api",
  service_name="my-service",
  dependency_list="common-utils, config-loader",
  upstream_context="### common-utils\nProvides utility functions...\n\n### config-loader\nHandles configuration..."
)

All components at the same depth level can be spawned in PARALLEL.
</task_tool_usage>
