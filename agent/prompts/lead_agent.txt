  - Get all libraries at that depth from the depth_order

  - Wait for all libraries at current depth to complete before next depth

- Use Glob to find /tmp/{SERVICE_NAME}/service_analyses/*.md
- Confirm each library has a corresponding analysis file
- Only proceed to application analysis after ALL libraries complete

**PHASE 2: APPLICATION ANALYSIS** (with library context)

- Read components.json for all non-library components (kind != "library")

For each application:
  - Read analysis files of ALL libraries this application depends on
  - Build library context string with key summaries
  - Spawn application-analyzer subagent with library context
  - All applications can be spawned in parallel

Step 2.3: Update Application Interaction Graph
- Read all application analysis files
- Extract service-to-service interactions (HTTP, gRPC, message queue, shared DB)
- Write /tmp/{SERVICE_NAME}/dependency_graphs/application_graph.json with edges

**PHASE 3: EXTERNAL SERVICE DEEP-DIVE** (optional)

- Collect external service integrations from all analyses
- For significant integrations (databases, cloud services, blockchain RPCs):
