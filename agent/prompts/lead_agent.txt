Step 1.2: Classify Components as Applications or Libraries

**Applications** are executable systems with entrypoints and business purposes:
- Web servers, APIs, microservices
- CLI tools, command-line applications
- Background workers, daemons, agents
- Batch processors, ETL scripts

**Libraries** are reusable code modules without entrypoints:
- Utility libraries (common-utils, helpers)
- Data models and schemas
- Shared business logic
- Framework extensions

For Rust codebases:
- Use Glob: pattern="**/Cargo.toml", exclude target/ directories
- Read each Cargo.toml to extract:
  * Package name (in [package] section, line starts with "name =")
  * Description (line starts with "description =")
  * Dependencies (in [dependencies] section, lines with "path =" are internal)
- Identify component root (directory containing Cargo.toml)
- Note key files (src/main.rs, src/lib.rs, src/**/*.rs)
- Classify as APPLICATION if:
  * Cargo.toml contains [[bin]] section (check for exact text "[[bin]]")
  * OR src/main.rs file exists in component root
  * OR name contains: "server", "api", "cli", "service", "agent", "daemon", "worker"
- Classify as LIBRARY if:
  * Cargo.toml only has [lib] section (no [[bin]], no src/main.rs)
  * OR only src/lib.rs exists (no src/main.rs)
  * OR name contains: "lib", "core", "utils", "common", "client", "shared", "schemas", "models"
  * Code doesn't provide executable entrypoints
- Default: "library" if uncertain

For Node.js:
- Use Glob: pattern="**/package.json", exclude node_modules/
- Read package.json, parse JSON
- Classify as APPLICATION if:
  * Has "bin" field in package.json
  * Has "main" pointing to server/app file
  * OR name contains: "server", "api", "cli", "service"
- Classify as LIBRARY if:
  * No "bin" field
  * OR name contains: "lib", "utils", "common", "shared", "client"
- Internal deps have "file:" prefix in dependency spec

For Python:
- Use Glob: pattern="**/pyproject.toml"
- Read pyproject.toml
- Extract name and description
- Classify as APPLICATION if:
  * pyproject.toml has [project.scripts] section
  * OR __main__.py exists in package root
  * OR name contains: "server", "api", "cli", "agent", "daemon", "worker"
- Classify as LIBRARY if:
  * No [project.scripts] entry
  * OR name contains: "lib", "core", "utils", "common", "client", "shared", "schemas"
- Default: "library" if uncertain

**PHASE 2: DEPENDENCY ANALYSIS - BUILD TWO GRAPHS**

<discovery_patterns>
**Rust Component Discovery with Glob and Read:**

Step 1: Find all Cargo.toml files
```
Use Glob with pattern: "Cargo.toml"
OR
Use Bash: find . -name "Cargo.toml" -not -path "*/target/*"
```

Step 2: For each Cargo.toml, extract info and classify:
```
Use Read to load file content
Parse for:
- Line starting with 'name = "' → extract package name
- Line starting with 'description = "' → extract description
- In [dependencies] section, lines with 'path = "' → internal dependencies
- Check for [[bin]] section or src/main.rs → APPLICATION
- Otherwise → LIBRARY
```

Example Cargo.toml (Application):
```toml
[package]
name = "auth-service"
description = "User authentication HTTP service"

[[bin]]
name = "auth-service"
path = "src/main.rs"

[dependencies]
common-utils = { path = "../common-utils" }  ← LIBRARY (internal)
database-models = { path = "../database-models" }  ← LIBRARY (internal)
serde = "1.0"  ← EXTERNAL
tokio = "1.35"  ← EXTERNAL
```

Classification:
- Type: APPLICATION (has [[bin]] or src/main.rs)
- Name: "auth-service"
- Libraries Used: ["common-utils", "database-models"]
- External Deps: ["serde", "tokio"]

Example Cargo.toml (Library):
```toml
[package]
name = "common-utils"
description = "Shared utility functions"

[dependencies]
serde = "1.0"  ← EXTERNAL
```

Classification:
- Type: LIBRARY (no [[bin]], no src/main.rs, only src/lib.rs)
- Name: "common-utils"
- Library Dependencies: [] (no internal library dependencies)
- External Deps: ["serde"]

**Bash Commands for Discovery:**
```bash
# List all Cargo.toml locations
find . -name "Cargo.toml" -not -path "*/target/*"

# Quick package name check
grep -m 1 '^name = ' Cargo.toml | cut -d'"' -f2

# Check if it's an application (has main.rs)
test -f src/main.rs && echo "APPLICATION" || echo "LIBRARY"
```

<context_rules>
**CRITICAL: Direct Dependencies Only**

For Libraries:
If library C depends on [A, B], and B depends on [X]:
- When spawning analyzer for C, pass analyses of A and B ONLY
- Do NOT pass X's analysis, even though C transitively depends on X through B

For Applications:
If application MyApp uses libraries [Utils, Database], and Database depends on [Utils]:
- When spawning analyzer for MyApp, pass analyses of Utils and Database
- The analyzer can see that Database uses Utils by reading Database's analysis

Example:
```
common-utils (library, no deps)
    ↓
database-models (library) depends on [common-utils]
    ↓
auth-service (application) uses [common-utils, database-models]
    ↓ (runtime interaction)
user-service (application) - auth-service calls user-service APIs
```

When analyzing auth-service:
- Pass analyses of: common-utils, database-models (libraries it uses)
- Do NOT pass: user-service (that's a runtime interaction, discovered during analysis)

Why: This keeps context focused and manageable.
</context_rules>
