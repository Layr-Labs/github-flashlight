**IMPORTANT: D3.js Graph Best Practices**

For the DependencyGraph component, implement these features:

1. **Layout Algorithm**:
   - Use d3.forceSimulation for organic layout
   - OR use d3.hierarchy + d3.tree for hierarchical (Phase 1 at top, Phase 2 below)
   - Consider the two-phase nature (Phase 1 foundation, Phase 2 dependent services)

2. **Visual Encoding**:
   - Node size: proportional to number of key components or dependencies
   - Node color: by phase (Phase 1/Foundation vs Phase 2/Dependent)
   - Edge thickness: could indicate coupling strength (optional)
   - Edge arrows: show dependency direction

3. **Interactivity**:
   - Click node: navigate to service detail OR highlight dependencies
   - Hover node: show tooltip with service name, type, brief description
   - Drag nodes: reposition in force layout
   - Zoom/Pan: use d3.zoom() for graph exploration
   - Double-click: center on node and highlight its subgraph

4. **Filters**:
   - Toggle Phase 1 / Phase 2 visibility
   - Filter by service type (rust-crate, nodejs-package, etc.)
   - Show only dependencies of selected service

5. **Legend**:
   - Color legend for phases
   - Size legend if using size encoding
   - Symbol legend if using different shapes

Example implementation structure:
```javascript
import React, { useRef, useEffect, useState } from 'react';
import * as d3 from 'd3';

const DependencyGraph = ({ services, graph, onNodeClick }) => {
  const svgRef = useRef();
  const [dimensions, setDimensions] = useState({ width: 900, height: 600 });

  useEffect(() => {
    if (!graph) return;

    const svg = d3.select(svgRef.current);
    svg.selectAll('*').remove(); // Clear previous

    const width = dimensions.width;
    const height = dimensions.height;

    // Setup zoom
    const zoom = d3.zoom()
      .scaleExtent([0.1, 4])
      .on('zoom', (event) => {
        g.attr('transform', event.transform);
      });

    svg.call(zoom);

    const g = svg.append('g');

    // Create force simulation
    const simulation = d3.forceSimulation(graph.nodes)
      .force('link', d3.forceLink(graph.edges).id(d => d.id).distance(100))
      .force('charge', d3.forceManyBody().strength(-300))
      .force('center', d3.forceCenter(width / 2, height / 2))
      .force('collision', d3.forceCollide().radius(30));

    // Draw edges
    const link = g.append('g')
      .selectAll('line')
      .data(graph.edges)
      .join('line')
      .attr('stroke', '#999')
      .attr('stroke-opacity', 0.6)
      .attr('stroke-width', 2)
      .attr('marker-end', 'url(#arrowhead)');

    // Arrow marker
    svg.append('defs').append('marker')
      .attr('id', 'arrowhead')
      .attr('viewBox', '-0 -5 10 10')
      .attr('refX', 25)
      .attr('refY', 0)
      .attr('orient', 'auto')
      .attr('markerWidth', 8)
      .attr('markerHeight', 8)
      .append('svg:path')
      .attr('d', 'M 0,-5 L 10 ,0 L 0,5')
      .attr('fill', '#999');

    // Draw nodes
    const node = g.append('g')
      .selectAll('circle')
      .data(graph.nodes)
      .join('circle')
      .attr('r', 20)
      .attr('fill', d => d.phase === 'phase1' ? '#3b82f6' : '#10b981')
      .attr('stroke', '#fff')
      .attr('stroke-width', 2)
      .style('cursor', 'pointer')
      .call(d3.drag()
        .on('start', dragstarted)
        .on('drag', dragged)
        .on('end', dragended))
      .on('click', (event, d) => {
        if (onNodeClick) onNodeClick(d.id);
      })
      .on('mouseover', function(event, d) {
        // Tooltip logic
        d3.select(this).attr('r', 25);
      })
      .on('mouseout', function(event, d) {
        d3.select(this).attr('r', 20);
      });

    // Node labels
    const label = g.append('g')
      .selectAll('text')
      .data(graph.nodes)
      .join('text')
      .text(d => d.id)
      .attr('font-size', 12)
      .attr('dx', 25)
      .attr('dy', 5)
      .style('pointer-events', 'none');

    // Update positions on tick
    simulation.on('tick', () => {
      link
        .attr('x1', d => d.source.x)
        .attr('y1', d => d.source.y)
        .attr('x2', d => d.target.x)
        .attr('y2', d => d.target.y);

      node
        .attr('cx', d => d.x)
        .attr('cy', d => d.y);

      label
        .attr('x', d => d.x)
        .attr('y', d => d.y);
    });

    function dragstarted(event, d) {
      if (!event.active) simulation.alphaTarget(0.3).restart();
      d.fx = d.x;
      d.fy = d.y;
    }

    function dragged(event, d) {
      d.fx = event.x;
      d.fy = event.y;
    }

    function dragended(event, d) {
      if (!event.active) simulation.alphaTarget(0);
      d.fx = null;
      d.fy = null;
    }

    return () => simulation.stop();
  }, [graph, dimensions, onNodeClick]);

  return (
    <div>
      <svg
        ref={svgRef}
        width={dimensions.width}
        height={dimensions.height}
        style={{ border: '1px solid #ddd', borderRadius: '4px' }}
      />
      <div style={{ marginTop: '10px' }}>
```

**Positive Psychology UX Design Principles**

Apply these principles throughout the entire website to create a warm, encouraging experience:

**1. Progressive Disclosure with Encouragement**
- Don't overwhelm users with information
- Reveal complexity gradually with friendly guidance
- Use expanding sections with encouraging headers: "Want to learn more? ▼"
- Celebrate exploration: "Great! You're diving deeper ✨"

**2. Positive Feedback Loops**
- Acknowledge every user action: clicks, hovers, navigation
- Use micro-animations to confirm interactions
- Success states are celebratory: "Found it! 🎉"
- Even errors are opportunities: "Oops! Let's fix this together 🔧"

**3. Sense of Progress & Achievement**
- Show what users have explored: "You've checked out 3/10 services! 🌟"
- Breadcrumb trails show journey: Home → Services → Details
- Completion indicators for sections
- Optional: Achievement badges (view all services, explore graph, etc.)

**4. Social & Human Connection**
- Use "we" and "you" language: "Let's explore together"
- Conversational tone, not corporate/robotic
- Show personality through copy and design
- Credits/attribution feel personal: "Analyzed with ❤️ by AI assistants"

**5. Delight Through Surprise**
- Easter eggs for power users (keyboard shortcuts shown with ✨)
- Random encouraging messages: "You're doing great!" "Nice find!" "Excellent choice!"
- Playful 404 page: "Lost in the code? Let's get you back on track! 🧭"
- Hover surprises: gentle animations, reveals, particle effects

**6. Autonomy & Control**
- Give users control over their experience
- Theme toggle: Light/Dark with smooth transition
- Customize graph view, filters, sorting
- "Your preferences saved ✓" confirmation
- Clear undo/reset options

**7. Reduce Anxiety & Frustration**
- No harsh red errors - use warm oranges/ambers
- Loading states are reassuring, not stressful
- Empty states have clear next steps
- No dead ends - always provide a path forward
- Forgiving search: "Did you mean...?" suggestions

**8. Celebrate Milestones**
- First service viewed: "🎉 First exploration complete!"
- Graph fully loaded: "✨ All dependencies mapped!"
- API docs accessed: "📖 Diving into the details, nice!"
- Completion: "🎊 You've explored the entire codebase!"

**9. Beauty & Craft**
- Details matter: smooth animations, perfect alignment
- Consistent spacing and rhythm
- Quality over quantity in visual elements
- Polish the small things: hover states, transitions
- Show care in typography and color choices

**10. Growth Mindset Language**
- "Explore" not "View"
- "Discover" not "See"
- "Learn about" not "Read"
- "Understanding your code" not "Analyzing code"
- "Journey" not "Process"

**Example Friendly Messages Throughout:**
- Welcome: "👋 Hey there! Ready to explore your codebase?"
- Loading: "Hang tight, great things take a moment... ⚡"
- Empty search: "Hmm, nothing here yet. Try searching for something else! 🔍"
- Success: "Perfect! Found {X} services matching your search ✨"
- Navigation: "Taking you to {destination}... 🚀"
- Error: "Oh no! Something went wrong. Let's try that again 🔄"
- 404: "Oops! This page is hiding. Let's get you back home 🏠"
- Copy success: "Copied! ✓"
- API loading: "Fetching API docs... Almost there! 📖"

**Animation Guidelines:**
- Duration: 200-400ms (feels responsive, not slow)
- Easing: cubic-bezier(0.4, 0, 0.2, 1) or ease-in-out
- Entry animations: fade-in + slide up (20px)
- Exit animations: fade-out + slide up (10px)
- Hover: scale(1.02) or translateY(-4px)
- Loading: gentle pulse or rotating spinner with personality
- Success: quick bounce or scale effect
- Avoid jarring or overly bouncy animations

**Integrating OpenAPI/Swagger Documentation**

**When to Include:**
- For any service that has a corresponding *_openapi.yaml or *_openapi.json file
- These files are generated by the code analyzer for HTTP REST/web services

**Implementation Steps:**

1. **Data Loading:**
   - During STEP 1, glob for `*_openapi.yaml` and `*_openapi.json` files
   - Read each OpenAPI spec file
   - Match to services by filename (e.g., `auth-service_openapi.yaml` → `auth-service`)
   - Parse YAML using js-yaml if needed
   - Store the parsed spec object in the service's data

2. **Component Integration:**
   - Install swagger-ui-react package (already in package.json dependencies)
   - Create SwaggerUI.jsx component that accepts spec object as prop
   - Import swagger-ui-react CSS for proper styling
   - Component should handle case where spec is null/undefined

3. **Routing:**
   - Add route: `/services/:serviceName/api` for API documentation page
   - This route renders the SwaggerUI component with the service's OpenAPI spec
   - Include "Back to Service" navigation link

4. **Service Detail Enhancement:**
   - In ServiceDetail.jsx, check if `service.hasApiDocs` is true
   - If true, show an "API Documentation" button/link
   - Link to `/services/:serviceName/api`
   - Can also show a preview of endpoint count: "5 HTTP endpoints documented"

5. **Data Structure:**
   ```javascript
   {
     name: "auth-service",
     hasApiDocs: true,
     openApiSpec: {
       openapi: "3.0.0",
       info: { title: "Auth Service API", version: "1.0.0" },
       paths: { /* ... */ },
       components: { schemas: { /* ... */ } }
     }
   }
   ```

**Benefits:**
- Users can explore HTTP APIs interactively
- Complete API contracts are documented
- Request/response schemas are visible
- Can test endpoints directly from the browser (if enabled)
- Complements the service analysis with concrete API definitions
