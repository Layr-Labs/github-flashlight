<edge_transformation>
**CRITICAL: Edge Property Transformation for D3.js**

The dependency graph JSON files use different property names than D3.js expects:

**Source Format** (in JSON files):
```json
{
  "edges": [
    {"from": "serviceA", "to": "serviceB", "type": "grpc", "description": "..."}
  ]
}
```

**Target Format** (for D3.js visualization):
```javascript
{
  edges: [
    {source: "serviceA", target: "serviceB", type: "grpc", description: "..."}
  ]
}
```

**Transformation Rule**:
- When copying edges from dependency_graph.json, application_graph.json, or library_graph.json:
  1. Change "from" property to "source"
  2. Change "to" property to "target"
  3. Keep ALL other properties unchanged (type, description, bidirectional, etc.)

**Example Transformation**:
```javascript
// Input from JSON file:
{"from": "coordinator", "to": "api", "type": "shared_database", "description": "...", "bidirectional": true}

// Output in analysisData.js:
{source: "coordinator", target: "api", type: "shared_database", description: "...", bidirectional: true}
```

This transformation is REQUIRED for all three graph types:
- dependencyGraph (library dependency graph)
- applicationGraph (application interaction graph)
- libraryGraph (if separate library graph exists)
</edge_transformation>
